# Stage Tabs — تبويبات المراحل

**التاريخ:** 2026-04-09
**الهدف:** تبويبات علوية فخمة (Google-style) لكل مرحلة مفعّلة، مع عزل كامل بين المراحل

---

## 1. الشكل العام

تبويبات في الهيدر — المساحة الفاضية بين اللوقو (يمين) واسم المستخدم (يسار):

```
┌──────────────────────────────────────────────────────────────┐
│  خروج │ مدير │ admin │ 🔔  │  ◉ ابتدائي │ متوسط │ ثانوي  │  🏫 نظام شؤون الطلاب │
└──────────────────────────────────────────────────────────────┘
```

### التصميم البصري (Google-style)
- التبويب المختار: خط سفلي سميك (3px) بلون `#4f46e5` + نص بنفس اللون + font-weight: 700
- باقي التبويبات: نص رمادي `#6b7280` + font-weight: 500
- Hover: خلفية خفيفة `#f3f4f6` + border-radius
- انتقال سلس: `transition: all 0.2s ease`
- لا يوجد تبويب "الكل" — كل مرحلة مستقلة تماماً

---

## 2. قواعد العرض

| الحالة | النتيجة |
|--------|---------|
| Admin + مرحلتين أو أكثر مفعّلة | تظهر التبويبات في الهيدر |
| Admin + مرحلة واحدة فقط | لا تظهر تبويبات — يدخل مباشرة لمرحلته الوحيدة |
| Deputy (وكيل) | لا تظهر — مرحلته مثبتة تلقائياً من `scopeValue` |
| Counselor / Guard / Teacher / أي دور آخر | لا تظهر |

---

## 3. السلوك

### الافتراضي
- أول مرحلة مفعّلة (الأقل ترتيباً: Kindergarten → Primary → Intermediate → Secondary) تكون مختارة تلقائياً
- بدون شاشة اختيار — يدخل مباشرة

### الثبات
- المرحلة المختارة تبقى ثابتة عند التنقل بين الصفحات
- تُحفظ في `localStorage` (key: `selectedStage`) — لما يسكّر المتصفح ويفتحه مرة ثانية، يرجع لنفس المرحلة

### العزل الكامل
- كل البيانات (طلاب، مخالفات، غياب، تأخر، استئذان، ملاحظات، سلوك متمايز، تقارير...) تُفلتر بالمرحلة المختارة
- لا يوجد خلط بيانات بين المراحل أبداً

### فلاتر المراحل القديمة
- تُحذف بالكامل من كل الصفحات (FilterBtn للمراحل)
- التبويب العلوي يحل محلها
- أي مكان كان يعرض فلتر مرحلة داخل الصفحة → يُزال

---

## 4. الصفحات المستثناة

- **الإعدادات** (`/settings`): لا تتأثر بالمرحلة — المدير يرى كل الإعدادات كاملة مهما كانت المرحلة المختارة

---

## 5. ترتيب المنسدلات (Dropdowns)

كل المنسدلات في التطبيق (طلاب، فصول، صفوف) تتبع هذا الترتيب:

```
الصف → الفصول (أ، ب، ج...) → الصف التالي
```

مثال كامل:
```
الأول ابتدائي / أ
الأول ابتدائي / ب
الأول ابتدائي / ج
الثاني ابتدائي / أ
الثاني ابتدائي / ب
...
السادس ابتدائي / ج
الأول متوسط / أ
الأول متوسط / ب
...
الثالث ثانوي / ج
```

عند اختيار مرحلة معينة من التبويبات، المنسدلات تعرض فقط صفوف وفصول تلك المرحلة.

---

## 6. التنفيذ التقني

### المكونات الجديدة
- `StageTabs` component — يُعرض في الهيدر (`AuthenticatedLayout`)

### التعديلات على المكونات الموجودة
- `AppContext` — إضافة `activeStage` + `setActiveStage` للحالة المركزية
- `AuthenticatedLayout` (App.tsx) — عرض `StageTabs` في الهيدر
- `usePageData` hook — استخدام `activeStage` من Context بدل `stageFilter` المحلي
- كل الصفحات — إزالة فلاتر المراحل المحلية (FilterBtn)
- المنسدلات (StudentSelector وغيرها) — فلترة بالمرحلة المختارة

### تدفق البيانات
```
StageTabs (هيدر) → setActiveStage → AppContext
                                        ↓
                              usePageData يقرأ activeStage
                                        ↓
                              كل الصفحات تُفلتر تلقائياً
```

---

## 7. حالات خاصة

- **مرحلة واحدة مفعّلة:** لا تبويبات، المرحلة مثبتة تلقائياً
- **المرحلة المحفوظة في localStorage لم تعد مفعّلة:** يرجع للمرحلة الافتراضية (الأقل ترتيباً)
- **الوكيل:** المرحلة تُحدد من `user.scopeValue` — لا تبويبات ولا خيار تغيير
